How to deploy
  1. In Railway, open this service → click Variables → add the required vars below.
  2. For BETTER_AUTH_SECRET set the value to ${{secret(32)}} — Railway generates it automatically.
  3. For DATABASE_URL use ${{Postgres.DATABASE_URL}} to reference your Postgres service.
  4. Click Refresh below until all required vars are green, then click Launch Paperclip.
  5. Switch to the Manage tab — copy the bootstrap invite link and register your admin account.
  6. Once registered, set PAPERCLIP_AUTH_DISABLE_SIGN_UP=true in Railway Variables to lock down signups.

⚠️ Reset setup

Deletes config.json and the saved invite link from the volume, then stops Paperclip. On next boot, the setup flow runs again.

This does NOT delete your database. All users, companies, and data in Postgres remain intact. To fully start over, delete the Postgres service in Railway and add a new one.
